# Taller de : personalización de etiquetas y fechas límite

**RECORDATORIO TEÓRICO** Trello organiza el trabajo mediante tableros, listas y tarjetas. Sin embargo, para gestionar proyectos de marketing digital de forma realmente operativa, dos funcionalidades resultan esenciales: las etiquetas (labels) y las fechas límite (due dates). Las etiquetas son códigos visuales de color que permiten categorizar las tarjetas según su naturaleza, prioridad, canal o estado. Por ejemplo, puedes usar el color rojo para tareas urgentes, el verde para contenidos listos para publicar y el azul para acciones vinculadas a redes sociales. Las fechas límite, por su parte, asignan un plazo concreto a cada tarea, lo que activa alertas automáticas en Trello cuando la fecha se acerca o se supera. Un error frecuente entre los principiantes es crear etiquetas sin un criterio claro o coherente, lo que genera confusión visual en lugar de claridad. Otro error habitual es asignar fechas límite sin tener en cuenta las dependencias entre tareas. El modelo mental que debes interiorizar es el siguiente: cada tarjeta debe responder a tres preguntas: ¿qué tipo de acción es? (etiqueta), ¿cuándo debe estar lista? (fecha límite) y ¿quién es responsable? (asignación). Un ejemplo concreto: en una campaña de lanzamiento de producto, una tarjeta llamada «Redactar post para Instagram» podría llevar la etiqueta «Redes sociales» en azul y una fecha límite fijada 48 horas antes de la publicación prevista. **ENUNCIADO DEL TALLER** En este taller, trabajarás directamente sobre el tablero de «Campaña digital» que creaste en el taller anterior de Trello.
